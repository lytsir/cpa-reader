## 四步结构（死结构，缺一不可）

### 第一步：这段到底在讲什么？

**要求**：
- 基于原文，不是一句话带过，是**基于原文内容的1-2段概括**
- 回答核心问题：这部分内容到底在讲什么？结论是什么？
- 让小白读完知道"我要学什么"
- 严禁脱离原文编造，严禁用"一句话概括"敷衍

**错误示例**：
```
一句话：无形资产报废是因为它已经完全不能给企业带来经济利益...
```
**正确示例**：
```
这段讲的是无形资产报废的账务处理。报废和减值不同，减值是资产还能用但价值下降了，报废是资产已经完全不能用了。报废时要将无形资产的账面价值全部转销，差额计入营业外支出。这里的关键是：报废要同时冲销累计摊销、减值准备和无形资产原值，三步分录缺一不可。
```

---

### 第二步：你可能会卡在哪里？

**要求**：
- **必须从原文出发**，找出小白会真实卡住的地方
- 不能凭空捏造卡点，不能为了凑数编问题
- 每个卡点必须是原文中确实存在的难点
- 用 `- ` 列表列出2-4个，口语化，带问号

**错误示例**：
```
- 1. 资产负债表无形资产项目怎么填数？  （这不是卡点，是自问自答）
```
**正确示例**：
```
- 报废和减值到底有什么区别？容易搞混
- 报废时减值准备怎么处理？要和减值测试分开吗？
- 分录里累计摊销和减值准备都借方，无形资产贷方，方向容易记反
```

---

### 第三步：逐一破解

**要求**：
- 每个卡点**独立解析**，用 `卡点N：标题` 分块
- **详细到让小白看懂没压力**：每个概念掰开揉碎讲
- **逻辑清晰**：为什么→是什么→怎么做，顺序不能乱
- **具象化**：用原文场景中的数字、案例、分录来说明
- **严禁编故事**：不要用"老王借钱""小李炒股"等虚构场景
- **比喻基于原文**：用原文里提到的业务场景做比喻，一两句话点透
- **分录必须有经济含义**：借方记什么、贷方记什么、为什么这样记

**错误示例**（敷衍式）：
```
卡点1：报废是彻底没用，减值是还能用但值钱了
。减值是资产还能带来经济利益，只是价值下降了...
```
**正确示例**（详细解析）：
```
卡点1：报废和减值到底有什么区别？

减值是"还能用但不值钱了"——资产还能带来经济利益，只是账面价值高于可收回金额，需要计提减值准备。比如一项专利还能用3年，但市场上出现了更好的替代技术，它的可收回金额只有原来的一半，这时要减值。

报废是"彻底不能用了"——资产已经完全不能带来经济利益，不再符合资产定义。比如专利权到期了，法律保护没了，任何人都能免费用这项技术，它对企业来说就一文不值了，必须全部转销。

关键区别：减值后资产还在账上，只是账面价值降低了；报废后资产从账上彻底消失。
```

### 第四步：例题/表格/分录 详尽解读

**要求**：
- 原文中的**每个例题**必须四块拆解，不能跳过
- 原文中的**每个表格**必须三块解读，不能跳过
- 原文中的**每个分录**必须讲清经济含义，不能跳过
- **每个数字怎么来的**必须逐步展示，不可跳步
- **分录为什么要这样做**必须解释清楚：借方记什么、贷方记什么、背后的经济业务逻辑

#### 例题四块拆解（死格式）

```
**【例X-X】**

**条件翻译：**
把题干翻译成大白话。说明已知条件（给了什么数字、什么场景）和求解目标（要求算什么）。

**思路拆解：**
分步说明解题思路。第一步做什么、第二步做什么、用什么准则/公式。

**数字推导：**
逐步计算，金额一律阿拉伯数字。每一步都要有数字，不可跳步。
格式：算式 = 结果（单位）

**陷阱提示：**
指出常见错误：容易混淆的概念、容易漏掉的条件、容易算错的数字。至少1个，最好2-3个。
```

#### 表格三块解读（死格式）

```
**【表X-X】**

**表格全貌：**
整体说明表格在讲什么。几行几列、核心结论一句话概括。

**逐行解读：**
至少解读3个数据点。每行说明：这个数字代表什么、为什么是这个数、和其他行什么关系。
格式：
1. 第1行（XX）：……
2. 第2行（XX）：……
3. 第3行（XX）：……

**纵向逻辑：**
指出数据间的因果关系或趋势。为什么A行比B行高？表格整体想说明什么道理？
```

#### 分录详解（死格式）

```
借：科目名（顶格，不加引号）          金额（右对齐）
    科目名（顶格对齐）                 金额（右对齐）
    贷：科目名（缩进两格）             金额（右对齐）
        科目名（缩进对齐）             金额（右对齐）
```

**分录解释要求**：
- 不是模板化套话（"借方记资产增加"）
- 是针对**具体业务**的解释：这个分录反映什么业务，每个科目为什么在这个方向
- 示例：
```
解释：这一步把固定资产账面价值转入清理。借方"固定资产清理"记账面净值50万（原值85万-折旧30万-减值5万），"累计折旧"和"减值准备"冲销原有余额，贷方"固定资产"冲销原值85万。清理完成后，固定资产从账上消失，后续处置损益都在"固定资产清理"里核算。
```

## 核心铁律（违反任何一条视为不合格）

1. **基于原文，严禁编造**：所有内容必须基于原文，不能脱离教材编故事、编案例
2. **详细到小白看懂**：不能敷衍，每个概念必须掰开揉碎讲
3. **逻辑链条完整**：why→what→how，不能跳跃
4. **具象化但不编故事**：用原文场景中的数字和案例，不用虚构人物
5. **例题/表格/分录不跳过**：原文有就必须详尽拆解
6. **每个数字讲清楚来源**：不可跳步，不可直接给答案
7. **分录解释针对具体业务**：不能用"借方记增加"这种通用模板
8. **禁用词零容忍**：讲真、说白了、你要知道、简单说、上述、计算得出、如表所示、根据相关规定
9. **中文数字零容忍（金额）**：所有金额、数量、百分比用阿拉伯数字
10. **格式四块结构**：`## 这段到底在讲什么？` / `## 你可能会卡在这里` / `## 逐一破解` / 例题详解

## 执行方式（第三阶段）

**拒绝子代理批量生成**。第三阶段必须人工逐篇处理：

```
读取原文 → 理解核心 → 找出真实卡点 → 按四步编写 → 自检 → 保存
```

- 每篇处理前必须**逐字阅读原文**
- 每篇处理后进行**自检**（禁用词/中文数字/分录格式/内容深度）
- 全量扫描只在**章完成后**做最终验证
- 发现不合格立即重写，不可批量脚本修复
