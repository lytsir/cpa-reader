# CPA三栏阅读器大白话生成黄金法则 v3 FINAL

## 一、宪法（不可违反的铁律）

### 铁律1：格式四块结构
所有大白话必须严格按以下四块结构输出，包括节级锚点和具体锚点，无一例外：

```
## 这段到底在讲什么？
[1-2段概括。节级锚点可稍长（3-4段），具体锚点1-2段。]

## 你可能会卡在这里
- 问题1？（不带引号，口语化问题）
- 问题2？
- 问题3？

## 逐一破解

卡点1：标题（不超过15字，概括核心）
[解析内容。禁止以"你可能会问""这里容易晕"开头。]

卡点2：标题
[解析内容]

卡点3：标题
[解析内容]
```

**格式细节**：
- `##` 后必须空一行再写内容
- "你可能会卡在这里"下必须用 `- ` 列表，严禁段落式引导语（"你可能会问：""这里容易晕：""不少人会搞混："）
- "逐一破解"下必须用 `卡点N：` 分块，严禁 `###` 子标题，严禁中文数字（"卡点一"）
- 每个卡点后空一行再写解析
- 节级锚点也需四块结构，不可写成散文

### 铁律2：禁用词零容忍
以下词汇在任何位置出现均视为致命错误：

```
讲真、说白了、你要知道、简单说、主要规范、上述、计算得出
如表所示、根据相关规定、阶段一/二/三/四、难点扫描
具象化方案、小白卡点、首先/其次/最后（正文语境）
```

**替换词库**：
- "说白了" → "也就是说"
- "讲真" → "实际上"
- "你要知道" → 删除，直接陈述
- "简单说" → "概括来说"
- "上述" → "前面提到的"
- "计算得出" → 直接写数字
- "如表所示" → "从表中可以看出"

### 铁律3：中文数字零容忍（金额）
所有金额、数量、百分比、年限必须用阿拉伯数字：
- ✅ "6,273万元"、"227万元"、"500万"、"10%"、"3年"
- ❌ "六千二百七十三万"、"二百二十七万"、"百分之十"、"三年"

**例外**：成语、固定搭配可保留中文数字（"铁板一块"、"混在一块"、"三番五次"）

### 铁律4：分录格式铁律
所有会计分录必须严格：
```
借：科目名（顶格，不加引号）          金额（右对齐）
    贷：科目名（缩进两格）              金额（右对齐）
```
- 借在上、贷缩进
- 科目名不加引号（❌ "固定资产清理" → ✅ 固定资产清理）
- 同方向多科目只写一个"借："，后续科目顶格对齐
- 金额必须右对齐（用空格补足）
- 分录必须有经济含义解释（借方记什么、贷方记什么）

### 铁律5：非结构性加粗零容忍
正文中严禁使用 `**加粗**` 强调。`**` 仅允许用于以下结构性标记：
- `**条件翻译：**`、`**思路拆解：**`、`**数字推导：**`、`**陷阱提示：**`
- `**表格全貌：**`、`**逐行解读：**`、`**纵向逻辑：**`
- `**借：**`、`**贷：**`
- `**【例X-X】**`、`**【题干】**`、`**【答案】**`
- 分录中的金额可不用加粗

### 铁律6：自检报告不入页面
自检报告仅用于生成者自查，严禁进入索引和页面。合并脚本必须自动剥离：
- `【自检报告】` 之后所有内容
- `自检报告` 之后所有内容
- `---自检报告` 之后所有内容

### 铁律7：Markdown表格不入页面
严禁使用 `|` Markdown表格。所有表格必须转为文字列表：
- ❌ `| A | B |` → ✅ `1. A——B`
- ❌ `| A | B | C |` → ✅ `1. A：B，C`

### 铁律8：虚构人物零容忍
严禁使用：老王、小李、小张、小明、小红等虚构人物。
比喻必须基于原文场景，禁止编故事。

### 铁律9：英文零容忍（正文）
正文严禁出现英文单词（专有名词如"IFRS"、"US GAAP"除外）。
术语首次出现需括号通俗解释。

### 铁律10：节级锚点=具体锚点
节级锚点（如"会计-第X章-第Y节"）和具体锚点（如"会计-第X章-第Y节-具体概念"）都必须按v3四步标准生成，不可写成散文式概述。

## 二、黄金法则（执行标准）

### 生成阶段

1. **读取原文**
   - 原文必须逐字读取，不可跳过
   - 标记原文中的例题编号（【例X-X】）、表格编号（表X-X）

2. **诊断难点**
   - 识别3-5个初学者最容易卡住的地方
   - 难点必须基于原文内容，不可凭空捏造

3. **设计具象化方案**
   - 比喻必须基于原文场景
   - 一两句话点透，禁止编故事
   - 禁止虚构人物

4. **编写四块内容**
   - 核心提炼：开篇"这段到底在讲什么？"，一句话概括核心
   - 卡点引导：用 `- ` 列表列出2-4个卡点问题
   - 逐一破解：每个卡点独立解析，用"卡点N："分块
   - 例题/表格/分录详解：如有必须按标准拆解

5. **例题四块拆解（如有）**
   - `**条件翻译：**` 把题干翻译成大白话
   - `**思路拆解：**` 分步说明解题思路
   - `**数字推导：**` 逐步计算，金额用阿拉伯数字
   - `**陷阱提示：**` 指出常见错误

6. **表格三块解读（如有）**
   - `**表格全貌：**` 整体说明表格在讲什么
   - `**逐行解读：**` 至少解读3个数据点
   - `**纵向逻辑：**` 指出数据间的因果关系或趋势

7. **自检**
   - 检查禁用词（逐词扫描）
   - 检查中文数字（金额/数量/百分比）
   - 检查分录格式
   - 检查非结构性加粗
   - 检查虚构人物
   - 检查篇幅（≥原文80%）

```python
def hard_check(aid, text):
    errors = []
    
    # 1. 结构检查
    if '## 这段到底在讲什么？' not in text:
        errors.append('缺"## 这段到底在讲什么？"')
    if '## 你可能会卡在这里' not in text:
        errors.append('缺"## 你可能会卡在这里"')
    if '## 逐一破解' not in text:
        errors.append('缺"## 逐一破解"')
    
    # 2. 格式检查
    if '### ' in text:
        errors.append('使用了###子标题（应改为卡点N：）')
    if re.search(r'卡点[一二三四五六七八九十][：:]', text):
        errors.append('中文数字卡点（应改为阿拉伯数字）')
    
    # 3. 禁用词检查
    banned = ['讲真', '说白了', '你要知道', '简单说', '主要规范', '计算得出', '如表所示', '根据相关规定']
    for word in banned:
        if word in text:
            errors.append(f'禁用词：{word}')
    
    # 4. 中文数字检查（金额）
    chinese_nums = re.findall(r'[一二两三四五六七八九十百千万亿]+(?:万|亿|千|百)(?:元|股|份|户|张|笔)?', text)
    real = [n for n in chinese_nums if len(n) >= 3 and any(u in n for u in ['万', '亿', '千', '百'])]
    if real:
        errors.append(f'中文数字金额：{real}')
    
    # 5. 非结构性加粗检查
    structural = ['条件翻译：', '思路拆解：', '数字推导：', '陷阱提示：',
                  '表格全貌：', '逐行解读：', '纵向逻辑：', '借：', '贷：',
                  '【题干】', '【答案】', '【分录】', '【拆解】']
    bold_matches = re.findall(r'\*\*([^*]{2,80})\*\*', text)
    for bm in bold_matches:
        is_structural = any(bm.strip().startswith(s) for s in structural) or re.match(r'【例\d+-\d+】', bm.strip())
        if not is_structural:
            errors.append(f'非结构性加粗：{bm}')
            break
    
    # 6. 虚构人物检查
    for person in ['老王', '小李', '小张', '小明', '小红']:
        if person in text:
            errors.append(f'虚构人物：{person}')
            break
    
    # 7. 分录格式检查
    # 检测科目名是否加引号
    if re.search(r'[""][\u4e00-\u9fa5]+(?:清理|折旧|准备|损益)[""]', text):
        errors.append('分录科目名加引号')
    
    # 8. 自检报告残留检查
    if '【自检报告】' in text or ('自检报告' in text and '阶段标记' in text):
        errors.append('自检报告残留')
    
    # 9. Markdown表格检查
    table_lines = [l for l in text.split('\n') if l.strip().startswith('|') and l.strip().endswith('|')]
    if len(table_lines) >= 3:
        errors.append('存在Markdown表格')
    
    # 10. 段落式引导语检查
    if re.search(r'^(你可能会问|这里容易晕|不少人会搞混|你可能会漏掉)[，,：:]', text, re.MULTILINE):
        errors.append('残留段落式引导语')
    
    return errors
```

## 三、累积经验清单（按章节）

### 第1章经验
- 子代理长文容易偷懒，后半篇质量下降，主代理必须抽查
- "会计工作责任"等政策性内容需引用最新文件编号

### 第2章经验
- 存货成本计算方法（FIFO/加权平均/个别计价）需配合例题详解
- 期末计量（可变现净值）是考试重点，需重点拆解

### 第3章经验
- 固定资产折旧方法需标准分录示例
- 处置（固定资产清理）五步分录必须完整
- 盘盈（以前年度损益调整）vs 盘亏（营业外收支）需对比

### 第4章经验
- 研发支出资本化条件（5个条件）需逐条解析
- 无形资产摊销/减值/处置的逻辑链条需打通
- 数据资源披露是新考点，需单独强调

### 第13章经验
- 金融资产分类（SPPI测试）是难点，需用大白话解释现金流特征
- 嵌入式衍生工具需用"拆不拆"框架解析
- 金融负债和权益工具区分需用"固定换固定"口诀

### 通用经验
- 每章合并前必须全量扫描，不可抽样
- 子代理输出必须清洗后再合并
- 禁用词清单必须内置到prompt和扫描脚本中
- 中文数字必须用 `cn2an` 库批量转换
- 节级锚点和具体锚点同等对待，都需四步结构

## 四、执行流程（死流程）

```
生成 → 自检 → 保存到 temp_第X章/ → 全量扫描 → 修复 → 合并到索引 → 全量扫描 → 推送
```

**不可跳过的步骤**：
1. 每篇生成后必须自检（禁用词/中文数字/分录格式）
2. 合并前必须全量扫描（不可抽样）
3. 合并后必须再次全量扫描
4. 推送前必须确认0致命错误

**子代理任务设计**：
- 每批最多4篇，不可超过
- 必须附带原文HTML路径
- 必须输出到指定目录
- prompt中必须包含本黄金法则前十条铁律
