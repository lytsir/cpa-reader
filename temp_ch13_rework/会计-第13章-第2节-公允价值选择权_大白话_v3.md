锚点ID: 会计-第13章-第2节-公允价值选择权

## 这段到底在讲什么？

企业在记账时，如果一项资产按公允价值计量（价值波动进利润表），但与之配套的负债却按摊余成本计量（不随市场波动），利润表就会"两边不同步"——这叫"会计错配"。这段讲的是准则给企业的"救命稻草"：在初始确认时，可以主动把某些金融资产、金融负债或一组金融工具指定为"以公允价值计量且其变动计入当期损益"，让两边同步波动、报表更真实。但这个选择权不是想用就用，必须满足特定条件，而且一旦指定就终身不能反悔。

## 你可能会卡在这里

### 卡点1：什么是会计错配？为啥会出现？

会计错配大白话就是：**同一笔经济业务的两边，用了不同的记账方法，导致报表上看起来不匹配**。

举个例子：某银行发行债券筹了1亿元，拿这笔钱去放贷款。债券按摊余成本计量（账面价值基本不动），贷款却按公允价值计量（价值随市场波动进利润表）。结果市场利率一涨，贷款账面跌了500万元进利润表，债券那边纹丝不动。从经济实质上看，利率涨对两边的影响应该互相抵消，但账面上只显示一边的损失——这就是会计错配。

错配导致报表失真，让报表使用者（如投资者）看不懂企业真实的经营状况。公允价值选择权就是用来解决这个问题的。

### 卡点2：公允价值选择权怎么用？条件是什么？

这个选择权只能在**初始确认时**用，中途不能补。可以对单项资产、单项负债用，也可以对一组工具一起用。但必须满足下面两个条件之一：

**条件1：能够消除或显著减少会计错配。**

这是最常见的理由。准则给了3种典型场景：

- **场景A**：资产是FVTPL，配套负债却是摊余成本。比如上述例子，把债券指定为FVTPL，两边都随市场波动，错配消除。
- **场景B**：资产和负债承担相同风险，公允价值变动方向相反、趋于抵消，但只有部分是FVTPL。比如利率涨的时候资产跌、负债也跌（或一个涨一个跌），本来应该对冲，但因为计量方式不同，对冲效果看不出来。把所有相关工具都指定为FVTPL，错配消除。
- **场景C**：资产和负债都没被指定为FVTPL，套期会计用不上。比如某银行发行上市债券为一组特定贷款融资，债券和贷款的公允价值变动可以互相抵消，但银行经常发行和回购债券，很少买卖贷款。如果都按摊余成本计量，每次回购债券都要确认利得或损失，贷款那边不动，两边始终对不上。把贷款和债券都指定为FVTPL，问题迎刃而解。

**条件2：以公允价值为基础进行管理和业绩评价。**

企业有正式书面文件（如风险管理政策、投资策略文件），明确规定以公允价值为基础管理金融负债组合或资产加负债组合，并且内部向关键管理人员报告业绩时也按公允价值评价，就可以行使这个选择权。

注意：已经按规定分类为FVTPL的金融资产，不需要再用这个选择权（它本来就是FVTPL了）。这个条件强调的是"日常管理和评价方式"，不是组合里每个工具本身的性质。

### 卡点3：指定有哪些"硬规矩"？

1. **只能应用于金融工具整体，不能拆**。你不能只对债券的前3年行使选择权，后2年不用；也不能只对本金部分用，利息部分不用。
2. **可以在合理延迟后补指定**。实务中企业可能难以把所有相关资产和负债在同一时间指定。只要能在每项相关交易的初始确认时就予以指定，且预期剩下的交易将来会发生，指定可以有合理的延迟。
3. **一经指定，不得撤销**。哪怕当初造成错配的那个工具已经被终止确认了（卖掉了或还清了），指定也不能撤销。这是一个极其刚性的规定，企业必须慎重。

### 卡点4：套期会计和公允价值选择权有什么关系？

两者都是解决"两边波动不同步"问题的工具，但门槛不同。

- **套期会计**：门槛很高，要求套期关系"高度有效"（80%-125%），需要持续测试有效性，文档要求严格。满足条件时，套期工具和被套期项目的公允价值变动可以同步确认。
- **公允价值选择权**：门槛相对较低，只要满足上述两个条件之一即可，不需要持续测试有效性。

当套期会计的有效性难以达到要求时，公允价值选择权就是一个替代方案。

## 例题详解

### 例题：银行发行债券为贷款融资的错配处理（源自原文场景B/C）

**条件翻译**：
某银行通过发行上市债券为一组特定贷款提供融资。债券与贷款的公允价值变动可相互抵销。银行定期发行和回购该债券，但很少买卖该贷款。假设：
- 贷款账面价值：10000万元
- 债券账面价值：10000万元
- 市场利率上升1%后，贷款公允价值下跌至9500万元，债券公允价值下跌至9600万元

**思路拆解**：
这道题考的是"没有一项是FVTPL、套期会计用不上"的错配场景。贷款和债券都按摊余成本计量时，利率变动不会在账面上反映，但银行频繁回购债券会产生已实现损益，贷款那边却不动。指定为FVTPL后，两边公允价值变动同步进利润表，自然对冲。

**数字推导**：

未指定FVTPL时（均按摊余成本）：
- 贷款账面：10000万元（不动）
- 债券账面：10000万元（不动）
- 银行回购债券时：假设以9600万元回购账面10000万元的债券，确认利得400万元
- 贷款那边仍按10000万元计，没有任何反映
- 报表结果：显示400万元利得，但经济实质上利率上升对贷款有500万元负面影响，两边不匹配

指定为FVTPL后：
- 贷款公允价值变动：9500 - 10000 = -500万元（进利润表）
- 债券公允价值变动：9600 - 10000 = -400万元（进利润表）
- 合计影响：-900万元，反映了利率上升对整体投资组合的真实影响
- 经济实质：利率上升导致资产和负债价值同向下跌，报表上两边同步体现，错配消除

**陷阱提示**：
1. 不要误以为"公允价值变动相互抵销"意味着贷款跌500、债券涨500——这里是同向变动（都跌），因为资产和负债对利率的敏感性方向相同。
2. "定期发行和回购债券但很少买卖贷款"说明套期会计难以应用，因为被套期项目（贷款）很少发生交易，套期关系难以建立和维持。
3. 指定必须在初始确认时作出，不能在利率已经变动后再"补指定"。

### 对比表：两种条件的关键区别

| 对比项 | 条件1：消除会计错配 | 条件2：以公允价值管理 |
|--------|------------------|-------------------|
| 核心目的 | 解决两边计量方式不匹配 | 反映企业真实的管理方式 |
| 是否需要书面文件 | 不强制要求 | 必须有正式书面文件 |
| 典型场景 | 资产FVTPL+负债摊余成本 | 投资组合日常盯市管理 |
| 适用对象 | 资产、负债或组合 | 主要面向负债组合或混合组合 |
| 延迟指定 | 可以合理延迟 | 应在初始确认时指定 |

### 公允价值选择权使用规则表

| 规则 | 具体内容 |
|------|---------|
| 指定时点 | 初始确认时，不得延后补指定 |
| 指定对象 | 单项资产、单项负债或一组工具 |
| 指定范围 | 只能整体指定，不能拆分 |
| 撤销规定 | 一经指定，终身不得撤销 |
| 终止确认后 | 即使工具已终止确认，指定仍不可撤销 |
| 合理延迟 | 可在初始确认时对每项交易分别指定，允许合理延迟 |
