# CPA三栏阅读器 - 大白话生成宪法 v3

> 违反以下任何一条，该篇大白话即为不合格，必须推倒重写。

---

## 一、问题根因（血淋淋的教训）

1. **批量泛化提示 → 子代理后半程放水**
   - 一次性给多篇时，LLM会对后面的篇章采用"教材句式+口语词包装"的偷懒策略。

2. **比喻不具体 → 抽象概括死灰复燃**
   - 没有唯一指定比喻时，LLM会退化为"下定义+举例子"的教科书写法。

3. **禁用词模糊 → 阳奉阴违**
   - 只说"去AI味"，LLM会保留"一方面...另一方面"、"由此可见"、"对于...至关重要"等句式。

4. **未强制打乱论述顺序 → 教材直译**
   - LLM顺着教材段落翻译，导致大白话的因果链和节奏与教材雷同。

5. **生成与humanizer混在一起 → 内容让位于形式**
   - 子代理为省事，先生成压缩版摘要，再套口语词，结果信息密度不足、比喻干瘪。

6. **缺少自检声明 → 浑水摸鱼**
   - 子代理不执行实质自检，把明显违规的输出直接上交。

7. **教材案例照搬 → AI味最浓的偷懒**
   - LLM直接翻译教材里的"甲公司"、"某企业"案例，导致后半段完全沦为教材复读。

8. **括号解释排比 → 变体列表化**
   - LLM用"可靠性（就是...）、相关性（就是...）、可比性（就是...）"的相同句式连续解释多个概念，本质还是列表。

9. **比喻只用"比作" → 无比喻**
   - LLM写"会计比作医生"但全文没有"就像.../好比.../相当于..."的具体场景句，导致比喻悬空，后面回归抽象论述。

10. **"其实"滥用 → 新结构痕迹**
   - LLM为凑口语词密度，连续多段以"其实"开头，形成新的排比痕迹。

11. **自检声明造假 → 形式主义防线崩溃**
   - 子代理在自检声明里写"合格"，但输出里明显有禁用词和等长句。声明不可信，必须用Python硬校验。

## 二、生产流水线（五步，缺一不可）

### Step 0: 原文质检
- 确保每个 anchor 对应的原文片段完整，无截断。
- 确保表格、例题、公式不被 anchor 切断。如果被切断，退回分段环节重新处理。

### Step 1: 生成大白话初稿（单篇输入）
**输入**：单篇教材原文  
**输出**：大白话初稿

**强制约束（违反任何一条=不合格）**：
1. **必须指定一个强具象化比喻**，且该比喻要**贯穿全文**。禁止篇首用一个比喻、后面回到抽象论述。
2. **每个专业术语首次出现时，必须用括号给出通俗解释**。如：权责发生制（就是收入和费用该算在谁头上的规矩）。
3. **必须解释"为什么"而不仅是"是什么"**。每出现一个规则/结论，必须跟上"因为这样会导致..."、"否则就会..."等因果链。
4. **必须彻底打乱教材的论述顺序**。禁止顺着教材段落逐句翻译。要重新组织：可以先讲后果、再讲规则、再讲例子。
5. **字数 ≥ 原文 × min_ratio**。长文（>300字）min_ratio ≥ 0.8，中短文（100-300字）min_ratio ≥ 1.5，超短标题（<100字）min_ratio ≥ 8.0。禁止压缩成摘要。
6. **禁止直接搬运教材原句做"定义"**。不允许先把教材的专业定义抄一遍，再在后面加个"打个比方"。
7. **涉及原教材例题时，必须进行例题深度解读**。原文中如果出现[例X-X]及具体数字、表格、计算步骤，大白话必须：
   - **逐行解析原例题的数字和计算逻辑**，不能跳过任何一步。
   - **用通俗比喻把计算过程讲清楚**，每一步数字的变化都要解释"为什么"。
   - **严禁用虚构例子替代原例题**。原题是丁公司就必须解析丁公司的数据，原题是甲公司W7型机器就必须解析W7型机器的数据。比喻用来帮助理解计算，不是用来替换原题。
   - **会计分录必须逐条拆解**，借方贷方分别代表什么，金额怎么算出来的，都要说清楚。
8. **禁止教材案例照搬（非例题场景）**。在论述性内容中，严禁出现"甲公司"、"乙公司"、"某企业"、"例如甲"等教材原案例。必须用自己的比喻或虚构场景替代。
9. **禁止使用英文词汇**。大白话是面向中文读者的通俗解读，严禁出现任何英文单词（如straightforward、also、option、bug、NFT、routine等）。原文中已有的专业缩写（如W7型机器、PPP项目）可保留，但不得新增英文表达。
10. **会计分录必须使用标准借贷格式**。涉及会计分录时，必须按以下格式呈现，禁止借贷挤在一行：
   ```
   借：存货跌价准备                        750000
   　　贷：资产减值损失——存货减值损失    750000
   ```
   借方在上，贷方缩进，金额右对齐。分录后面必须逐条解释借方和贷方的含义。
11. **每章必须配套"本章会计分录"完整汇总**。每章大白话完成后，必须额外生成本章涉及的全部会计分录汇总表（JSON格式），要求：
    - **齐全性**：不仅包含教材原题中的分录，还要覆盖该章节全部业务场景（如外购/委托加工/跌价准备/盘盈盘亏/计划成本法等）。
    - **结构化**：每条分录必须包含 `title`（业务场景）、`debit[]`（借方科目+金额）、`credit[]`（贷方科目+金额）、`大白话`（逐条解释借贷含义）。
    - **分组归类**：按业务类型分组（如"一、外购存货"、"二、委托加工物资"），每组下设具体条目。
    - **与大白话联动**：大白话正文中涉及的分录，必须与汇总表中的条目保持完全一致。

### Step 2: Humanizer 风格淬炼
**输入**：大白话初稿  
**输出**：最终大白话

**强制约束（违反任何一条=不合格）**：
1. **绝对禁用词组**（出现即死刑）：
   - 序列词：`第一/第二/第三`、`首先/其次/最后`、`一是/二是/三是`、`一方面/另一方面`、`还有一个方面`。
   - 过渡词：`具体来说`、`综上所述`、`由此可见`、`值得注意的是`、`不容忽视`。
   - 套话：`起到了积极作用`、`对于...至关重要`、`真正服务好...的发展需要`、`有机统一`。
   - 教材腔定义：`是...的基石`、`是...的灵魂`、`主要规范`、`主要包括`。
2. **禁止列表化呈现**。即使是五个要点，也必须写成**一段流动的散文**。禁止排比段（连续两段以相同句式开头）。
3. **禁止括号解释排比**。严禁连续使用"X性（就是...）、Y性（就是...）、Z性（就是...）"的相同结构解释多个概念。每个概念的解释句式必须完全不同。
4. **禁止机械替换**。不允许把"应当"改成"应该"、"不得"改成"不能"，但整句结构仍照搬教材。
5. **句子长短必须交替**。要有3-8字的短句，也要有25-40字的长句。**禁止连续3句长度相差在±15%以内**。
6. **口语化衔接词必须出现**：每150字内至少出现一次`说白了`、`你要知道`、`问题是`、`讲真`、`不过`中的至少一个。
   - **"其实"限用**：全文出现不得超过3次，且不能连续两段都以"其实"开头。
   - **分布均匀**：不能开头200字内用5次，后面1000字一次没有。
7. **禁止 em dash（—）**，统一用逗号或句号。
9. **比喻具象化硬标准**：全文必须至少包含一个包含`比如`、`例如`、`相当于`、`就好比`、`就像`、`好比`中任意一词的具体场景句（如"就像医生手术前必须洗手消毒一样"）。仅说"会计比作医生"不算达标。
10. **所有数字必须用阿拉伯数字表示**。中文数字（如一、二、三、十、百、千、万、亿）在金额、数量、百分比、时间、计算过程中**一律转换为阿拉伯数字**。如：
    - "一百一十三万六千元" → "113.6万元"
    - "百分之三" → "3%"
    - "十八万三千六百五十三点三三元" → "183,653.33元"
    - "三分之一" → "1/3"
    - "每年折旧十八万三千六百五十三点三三元" → "每年折旧183,653.33元"
    - 仅成语（如"一五一十"）、条件词（如"万一"）、口语量词（如"一点点"）可保留中文。
    - 会计分录中的金额必须纯阿拉伯数字，禁止夹杂中文。

### Step 3: 子代理强制自检（声明+硬校验双保险）
子代理必须在最终输出前，运行以下自检并**明确写出声明**：

```
【自检声明】
1. 具象化比喻是否贯穿全文：[是/否]
2. 是否出现禁用词组：[列出/无]
3. 是否出现列表化/排比段/括号解释排比：[是/否]
4. 是否有连续3句长度相近：[是/否]
5. 字数是否达标（原文X字，大白话Y字，比例Z%）：[是/否]
6. 是否存在教材原句结构搬运或教材案例照搬：[是/否]
7. 结论：[合格/不合格，如不合格说明重写计划]
```

**但声明不可靠，子代理经常造假。所以必须同步执行 Step 4 的 Python 硬校验脚本，只有脚本也零报错，才算真正合格。**

### Step 4: Python 自动质检脚本（真正防线）
运行脚本检测以下违规项，**命中任何一条即返回重写，不得交付**：
- 禁用词组命中（正则匹配，注意排除"第三方"等自然用语）
- 列表符号：`（1）`、`（2）`、`1.`、`2.`、`①`、`②`
- 连续"应当"/"不得"/"不能"/"必须"/"要求"出现 ≥3 次
- em dash（—）存在
- 字数不达标（严格计算：大白话字数 / 原文字数 < min_ratio）
- 连续3句长度相差在±15%以内
- 无比喻：全篇无`比如`/`例如`/`相当于`/`就好比`/`就像`/`好比`
- 口语词密度不足：每150字内无`说白了`/`你要知道`/`问题是`/`讲真`/`不过`
- "其实"滥用：出现超过3次，或连续两段以"其实"开头
- 教材案例关键词：`甲公司`、`乙公司`、`例如甲`

### Step 5: 人工抽检
- 对每一章至少抽检3篇（开头、中间、结尾）。
- **重点抽查子代理批量处理时的后半段篇章**，此处最容易放水。
- 抽检标准：闭眼读，如果感觉"这像是把教材念了一遍"，即为不合格，整章退回。

## 三、对子代理的死亡威胁（提示词必加）

> 你的输出将被直接用于商业出版物。如果你偷懒，采用"教材结构+口语词包装"的方式应付，会被用户一眼识破并当场要求推倒重写。这会浪费大量算力和时间，对你没有任何好处。
>
> 所以，请严格按照上述五步执行。如果你发现某篇很难用比喻贯穿，请返回 Step 1 换一个比喻重新写，而不是放弃比喻、回归抽象概括。
>
> **特别注意：严禁把教材里的"甲公司"、"某企业"案例直接翻译过来。用户对这种教材案例照搬零容忍。**
>
> **自检声明不可信，必须配合 Python 硬校验脚本零报错才算合格。如果你声明"合格"但脚本检测出问题，你的输出会被直接拒绝并退回重写。**

## 四、关键经验（刻烟吸肺）

- **具象化是硬约束，不是软建议**。没有比喻=不及格。
- **比喻不能悬空**。仅写"会计比作医生"不够，必须有"就像医生手术前洗手消毒一样"的场景句。
- **打碎教材结构比"口语化"更重要**。顺着教材翻译，换再多词也是AI味。
- **禁用词必须具体到词组+死刑威慑**。LLM会不断试探边界。
- **长原文最容易偷懒压缩**。min_ratio 是硬下限，不是建议，且要人工抽查最长篇。
- **超短标题（<100字）最容易信息不足**。必须强制 min_ratio ≥ 8.0，大幅展开解释。
- **生成和 humanizer 必须物理分两步**。一步完成必导致内容空洞。
- **自检声明不可靠，Python硬校验才是真正的防线**。子代理会在声明里撒谎。
- **教材案例照搬是最高级偷懒**。比机械替换更难察觉，但用户一眼就能认出来。
- **"其实"正在成为新的AI结构痕迹**。限用3次，禁止连续两段开头。
