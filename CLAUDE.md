# CPA大白话格式铁律（宪法）

> **强制执行条款：每次开始CPA任务前，必须先读取本文件（CLAUDE.md）和 index-state-machine.json。未读不动手，违者自己承担质量事故后果。**

## 四步结构（必须严格执行）

### 第一步：##这段到底在讲什么？
- 基于原文全部内容，客观概述
- 不加个人见解，不扩展原文外知识

### 第二步：##你可能会卡在这里
- 必须用"- "列表格式
- 严禁"你可能会问："段落式引导语
- 列出真实的理解难点，不能凭空想象

### 第三步：##逐一破解
- 必须用"卡点1：""卡点2："分块
- 严禁###子标题或"卡点一"中文数字
- 每个卡点结构：why→what→how
- 具象化解读，逻辑清晰，让小白看懂没压力
- 严禁编故事，严禁原文外知识

### 第四步：例题/表格/分录详解
- 如有例题、表格或图片，必须详尽解读
- 每个数字必须解释怎么来的
- 分录必须解释为什么要这样做
- **结构铁律：例题、表格、分录必须从"逐一破解"中剥离，单独成为独立区块**，格式为 `##表X-X深度解析` 或 `##例题X-X深度解析`
- "逐一破解"的卡点必须严格与"你可能会卡在这里"一一对应，修正例题/表格时不得改动卡点编号

## 禁用词清单（零容忍）

以下词汇绝对禁止出现：
- 讲真
- 说白了
- 你要知道
- 简单说
- 上述
- 计算得出
- 如表所示
- 根据相关规定

## 数字铁律（零容忍）

- 所有金额必须用阿拉伯数字
- 禁止中文数字（如"二百二十七万"应改为"227万"）
- 金额单位统一用万/元，不混用

## 分录格式（严格）

```
借：应付账款          1,000,000
    贷：银行存款          1,000,000
```

- 借方顶格
- 贷方缩进
- 金额右对齐
- 科目不加引号
- 不用表格显示分录

## 内容红线（零容忍）

- 严禁编故事：所有解读必须基于原文，不能自己编造案例或情节
- 严禁原文外知识：不能引用教材以外的会计准则或规定
- 必须逐条/逐个审核判断：尤其是表格化分录和"表X-X"标题的表格
- 不能为省时间批量处理多章，必须逐节由LLM读取原文并插入语义锚点

## 重写定义（核心铁律）

- **"重写" = 逐篇读取原文HTML → 不参考任何旧文件 → 按v3标准从零生成**
- 严禁基于已有文件做格式修正或内容微调
- 旧文件只能用于确认锚点ID是否存在，不能用于参考内容

## 自检脚本规范

金额中文数字检测须排除：
- 序数词："第一""第二""第三"
- 分点词："一是""二是""三是"
- 阿拉伯数字+单位："500万元""10,000元"中的"万元""元"

只匹配纯中文数字金额：如"一百万元""二千五百元"

## 批量处理上限

- 每会话处理不超过5篇
- 每篇生成后单独自检、单独保存
- 严禁一次性写入多篇后再统一自检

## 索引与文件名铁律（新增，零容忍）

1. **索引键名必须严格等于 HTML 锚点 ID**：如 `会计-第4章-第1节-无形资产的定义与基本特征`，严禁带 `_大白话`、`_dabaihua` 等任何后缀
2. **大白话文件必须是 `.md` 格式**：output 目录中严禁留存 `.txt` 或带 `_dabaihua`、`_大白话` 后缀的文件，sync 脚本只扫描 `.md`
3. **文件名必须严格等于锚点 ID**：如 `会计-第4章-第1节-无形资产的定义与基本特征.md`，特殊字符（逗号、顿号等）保留原样，不得替换为下划线
4. **每次生成前清理旧文件**：`find output/科目/ -name "*.txt" -o -name "*_dabaihua*" -o -name "*_大白话*"` 并删除
5. **推送前强制验证**：用脚本比对索引中的键名和 HTML 中的锚点 ID，必须 100% 匹配，少一个字都不行

## 原文确认流程

每篇处理前：
1. 读取该篇原文HTML
2. 展示给用户确认锚点ID和内容边界
3. 用户确认后再开始生成

## 状态机更新规范

- 修改前必须先 `read_file` 确认当前内容
- 严禁直接patch可能已变化的文件

## 执行顺序规范

每次会话必须按以下顺序：
1. 读取本文件（CLAUDE.md）
2. 读取状态机（index-state-machine.json）
3. 读取原文HTML
4. 按v3标准生成
5. 自检
6. 保存
7. 同步索引（`python3 scripts/sync-dahua-index.py --subject 科目名`）
8. GitHub提交推送

## 已知反复错误（每次生成前必须过一遍）

1. 中文数字 → 改为阿拉伯数字
2. 禁用词混入 → 删除或替换
3. "卡点一"中文数字 → 改为"卡点1："
4. 分录科目加引号 → 删除引号
5. 借方不顶格/贷方不缩进 → 修正对齐
6. 编故事/加原文外知识 → 删除，只保留原文解读
7. 批量处理多章跳过语义锚点 → 必须逐节处理
8. 索引与文件系统不同步 → 每次操作必须双向同步
9. 修改 output/ 目录 .md 后未同步 metadata JSON 索引 → 每次修改后必须执行 `python3 scripts/sync-dahua-index.py --subject 科目名` 后再提交
