# CPA大白话格式铁律（宪法）

## 四步结构（必须严格执行）

### 第一步：##这段到底在讲什么？
- 基于原文全部内容，客观概述
- 不加个人见解，不扩展原文外知识

### 第二步：##你可能会卡在这里
- 必须用"- "列表格式
- 严禁"你可能会问："段落式引导语
- 列出真实的理解难点，不能凭空想象

### 第三步：##逐一破解
- 必须用"卡点1：""卡点2："分块
- 严禁###子标题或"卡点一"中文数字
- 每个卡点结构：why→what→how
- 具象化解读，逻辑清晰，让小白看懂没压力
- 严禁编故事，严禁原文外知识

### 第四步：例题/表格/分录详解
- 如有例题、表格或图片，必须详尽解读
- 每个数字必须解释怎么来的
- 分录必须解释为什么要这样做

## 禁用词清单（零容忍）

以下词汇绝对禁止出现：
- 讲真
- 说白了
- 你要知道
- 简单说
- 上述
- 计算得出
- 如表所示
- 根据相关规定

## 数字铁律（零容忍）

- 所有金额必须用阿拉伯数字
- 禁止中文数字（如"二百二十七万"应改为"227万"）
- 金额单位统一用万/元，不混用

## 分录格式（严格）

```
借：应付账款          1,000,000
    贷：银行存款          1,000,000
```

- 借方顶格
- 贷方缩进
- 金额右对齐
- 科目不加引号
- 不用表格显示分录

## 内容红线（零容忍）

- 严禁编故事：所有解读必须基于原文，不能自己编造案例或情节
- 严禁原文外知识：不能引用教材以外的会计准则或规定
- 必须逐条/逐个审核判断：尤其是表格化分录和"表X-X"标题的表格
- 不能为省时间批量处理多章，必须逐节由LLM读取原文并插入语义锚点

## 已知反复错误（每次生成前必须过一遍）

1. 中文数字 → 改为阿拉伯数字
2. 禁用词混入 → 删除或替换
3. "卡点一"中文数字 → 改为"卡点1："
4. 分录科目加引号 → 删除引号
5. 借方不顶格/贷方不缩进 → 修正对齐
6. 编故事/加原文外知识 → 删除，只保留原文解读
7. 批量处理多章跳过语义锚点 → 必须逐节处理
8. 索引与文件系统不同步 → 每次操作必须双向同步
